🌐 HTML & CSS Practice Lab
Welcome to my personal playground for learning web development on Fedora. This repository tracking my progress from absolute basics (headings and text formatting) up to multimedia handling, forms, and basic CSS styling.

📂 Directory Roadmap & Files
📑 Basic HTML & Layouts
html/index.html / html/index2.html: The genesis. Messing around with titles, basic headings (<h1>-<h4>), lines breaks, and paragraphs.

html/txt_format.html: A deep dive into formatting tags (<b>, <i>, <del>, <sub/sup>), plus understanding structural blocks like <span> and <div>.

html/headers&footers.html: Setting up proper semantic page architecture using <header>, <main>, and <footer>.

🔗 Multimedia & Links
html/lyrics.html: A dedicated masterpiece for the legendary Said Mosker - Ana High Class. Integrates structural content with local images, audios, videos, and alternative <source> fallbacks.

html/H_links.html: Hyperlink practice covering external sites (Gemini), relative local files (lyrics.html), and anchor email triggers (mailto:).

html/imags.html / html/audio.html / html/video.html: Targeted experiments with interactive images, looping scream audios, moving clips (.mov/.webm), and standard media player control toggles.

html/favicon.html: Quick implementation test for customized tab icons (<link rel="icon">).

📊 Data & User Interaction
html/lists.html: Organized structure playground showcasing Unordered (<ul>), Ordered (<ol>), and Description (<dl>) layout patterns.

html/tables.html: Structured database view building a functional dark-themed "Store Hours" sheet.

html/button.html: A touch of interactivity. A custom styled button tied to a native JavaScript click event.

html/froms.html (forms): A full-scale data entry form including standard text inputs, date picks, phone patterns, dropdown selects, radios, file uploads, and submission flags.

🎨 CSS Integration
css/practice1/: Stepping out of pure markup. Features index.html linked cleanly to an external style.css sheet, utilizing target ID (#p1) and class (.odd/.even) rules.

🧪 Tech Toolkit Used
Markup: Semantic HTML5 (<video>, <audio>, <header>, <main>, <footer>)

Styling: Inline CSS & External CSS sheets

Logic: Basic Document Object Model (DOM) manipulation with JavaScript

Environment: Fedora Linux OS

📝 Key Rules Learned
Semantic tags (<header>, <main>) don't style your page dynamically on their own—they give meaning to the layout. CSS handles the aesthetic positioning.

Typos count: Kept a mental note to correct minor setup strings later (like target="_blanlk" to _blank).

Media Fallbacks: Always include alternative sources (.mp4 / .webm) inside media wrappers to ensure cross-browser compatibility.
